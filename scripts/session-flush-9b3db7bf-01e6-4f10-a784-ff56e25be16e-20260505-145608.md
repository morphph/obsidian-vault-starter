# Codex Subagents vs Claude Code Subagents：多智能体编程工作流对比

**TL;DR：** 两个平台都支持多智能体并行工作，但设计哲学截然不同。**Codex** 走云端沙箱路线，每个 subagent 运行在独立容器中，适合大规模批量任务和开源项目协作。**Claude Code** 走本地终端路线，subagent 共享项目上下文，适合需要深度代码理解的复杂重构任务。选择取决于你是要"广度并行"还是"深度协作"。

## 概述：Codex 的 Subagent 机制

[OpenAI Codex](/zh/blog/codex-complete-guide) 是 OpenAI 推出的云端编程智能体平台。它的多智能体模型基于独立沙箱容器——每个任务启动一个隔离环境，agent 在其中读取代码、执行命令、生成 diff。

Codex 的核心优势在于**异步并行**。你可以同时启动多个任务（修复 bug、写测试、重构模块），每个任务独立运行，互不干扰。这种架构特别适合开源项目维护者一次性处理多个 issue，或者团队批量执行代码迁移。

[Codex for Open Source](/zh/blog/codex-for-open-source) 计划让开源维护者免费使用这些能力，进一步降低了多智能体工作流的门槛。

## 概述：Claude Code 的 Subagent 机制

[Claude Code Agent Teams](/zh/blog/claude-code-agent-teams) 是 Anthropic 在 Claude Code 中实现的多智能体架构。与 Codex 不同，Claude Code 的 subagent 运行在同一个项目目录中，共享文件系统和 git 状态。

Claude Code 的 subagent 通过 `Agent` 工具调用产生，支持多种专业类型：`Explore`（只读搜索）、`codex-rescue`（诊断修复）、通用型等。主 agent 可以同时派遣多个 subagent 并行工作，然后汇总结果做出决策。这种"共享上下文"的设计让 subagent 之间的协作更紧密，适合需要跨文件理解的深度任务。

更多实际用例参见我们的 [Claude Code subagents 实战示例](/zh/blog/claude-code-subagents-examples)。

## 功能对比

| 维度 | Codex Subagents | Claude Code Subagents | 优势方 |
|------|----------------|----------------------|--------|
| **运行环境** | 云端独立沙箱容器 | 本地终端共享目录 | 各有所长 |
| **并行能力** | 多任务完全隔离并行 | 多 agent 共享上下文并行 | Codex |
| **上下文共享** | 每个 agent 独立读取仓库快照 | 共享项目文件系统和 git 状态 | Claude Code |
| **自定义 Agent** | 通过 system prompt 和工具配置 | SKILL.md + Agent 类型 + MCP | Claude Code |
| **任务粒度** | 适合独立、边界清晰的任务 | 适合需要跨文件协调的任务 | 各有所长 |
| **审批机制** | 生成 PR/diff 后人工审核 | 实时交互，逐步确认 | 各有所长 |
| **定价模型** | 按任务计费，含计算资源 | 按 token 计费 | 视用量而定 |
| **平台支持** | Web + VS Code 扩展 | 终端 + IDE 扩展 | Tie |

## 多智能体架构：设计哲学对比

两个平台对"subagent"的理解从根本上不同。

Codex 采用**任务分发模型**。你把一个大任务拆成多个独立子任务，分别交给不同的 agent 实例。每个实例在自己的沙箱中运行，看到的是仓库的一个冻结快照。这意味着 agent 之间不会互相干扰，但也无法实时协作。最终结果以 PR 形式呈现，由人类审核合并。

Claude Code 采用**团队协作模型**。主 agent 是"项目负责人"，它理解整体目标后，把具体的调研、搜索、实现任务分配给 subagent。subagent 完成后报告结果，主 agent 综合判断下一步。这种模式下，一个 subagent 的发现可以立即影响另一个 subagent 的工作方向。

对于[多智能体工作流](/zh/blog/con-u-pour-des-workflows-multi-agents)的实际落地，Codex 更像是"平行宇宙"——多条线独立推进；Claude Code 更像是"同一间办公室"——团队成员随时沟通。

## 自定义 Agent 能力对比

**Codex 的自定义方式：**

Codex 通过仓库中的配置文件（如 `AGENTS.md` 或 system prompt 模板）定义 agent 行为。你可以指定 agent 的角色、可用工具、代码风格约束。[Codex VS Code 扩展](/zh/blog/codex-vscode)让你在 IDE 中直接管理这些配置。

主要限制：自定义主要停留在 prompt 层面，工具链相对固定（读文件、执行命令、生成 diff）。

**Claude Code 的自定义方式：**

Claude Code 提供多层自定义：[SKILL.md 文件](/zh/blog/9-principles-writing-claude-code-skills)定义任务级指令，[MCP Server](/zh/glossary/agent-sdk) 扩展工具能力，Hooks 实现事件驱动的自动化。这套[扩展栈](/zh/blog/claude-code-extension-stack-skills-hooks-agents-mcp)让你能构建高度定制的 agent 工作流。

Claude Code 的 subagent 还支持类型化分工：只读搜索 agent 不会意外修改代码，rescue agent 专门处理卡住的任务。这种类型安全机制在 Codex 中没有对应设计。

## 适用场景：选择 Codex Subagents

以下场景 Codex 的多智能体方案更优：

- **批量处理开源 Issue**：一次启动 10 个 agent 分别处理不同的 bug report，每个生成独立 PR
- **大规模代码迁移**：把 200 个文件从旧 API 迁移到新 API，每个文件一个独立任务
- **CI 集成自动修复**：测试失败时自动启动 agent 尝试修复，成功则提交 PR
- **团队异步协作**：开发者提交任务后离开，agent 在云端完成工作，完成后通知

如果你的工作模式是"提交任务 → 去做别的事 → 回来审核结果"，Codex 的异步架构更匹配。[Codex for Students](/zh/blog/codex-for-students) 计划也降低了入门成本。

## 适用场景：选择 Claude Code Subagents

以下场景 Claude Code 的多智能体方案更优：

- **复杂重构**：需要理解多个模块间的依赖关系，subagent 调研后主 agent 制定统一方案
- **实时迭代开发**：写代码 → 发现问题 → 调整方向，需要 agent 之间实时同步状态
- **深度代码审查**：多个 subagent 分别检查安全、性能、风格，汇总成一份完整 review
- **探索性任务**：不确定最佳方案，派出多个 subagent 探索不同路径，根据结果选择

如果你的工作模式是"跟 AI 结对编程，边做边调"，Claude Code 的交互式协作更自然。参考 [Claude Code 不只是编程工具](/zh/blog/claude-code-is-not-a-coding-tool)的分析。

## 结论

**没有绝对优劣，只有场景匹配。** 如果你需要大规模并行处理边界清晰的独立任务，Codex 的沙箱隔离模型更高效、更安全。如果你需要多个 agent 协同完成一个复杂目标，Claude Code 的共享上下文模型提供更强的协作能力。

实际工程中，两者可以互补：用 Codex 批量处理标准化任务（迁移、格式化、简单 bug），用 Claude Code 处理需要判断力的复杂任务（架构决策、跨模块重构、技术方案选型）。关键是理解每个平台的[agentic coding](/zh/glossary/agentic-coding) 设计取舍，选对工具做对事。

## 常见问题

### Codex 的 subagent 能互相通信吗？

目前不能。Codex 的每个 agent 实例运行在独立沙箱中，看到的是仓库快照的只读副本。agent 之间没有直接通信通道，协调由人类通过审核 PR 来完成。

### Claude Code 最多能同时运行几个 subagent？

没有硬性上限，但受 context window 和 token 预算约束。实践中，同时派出 3-5 个 subagent 是常见模式，超过这个数量需要注意上下文管理。

### 两个平台的 subagent 都支持哪些编程语言？

两者都是语言无关的——只要你的项目能在终端/容器中运行，agent 就能处理。Codex 对 Python、JavaScript、TypeScript 的优化最明显；Claude Code 对所有主流语言支持均衡。

### 自定义 agent 需要写代码吗？

Codex 主要通过 prompt 配置，不需要写代码。Claude Code 的基础自定义（SKILL.md）也是纯文本，但高级功能（MCP Server、Hooks）需要编写 TypeScript/JSON 配置。
