**Use both — they solve different problems.** CLAUDE.md is your project's operating manual: deterministic, shared, version-controlled instructions that every Claude Code session follows. Claude Memory is your personal context layer: learned preferences, corrections, and temporary state that makes Claude work better for you specifically.

**Start with CLAUDE.md.** If you're setting up Claude Code for the first time, write a CLAUDE.md with your build commands, test requirements, and top three coding conventions. That alone will dramatically improve Claude's output. Memory will build itself naturally as you work — you don't need to configure it.

**For teams, CLAUDE.md is non-negotiable.** It's the only way to ensure consistent AI behavior across developers. Memory handles the personal customization that makes each developer's experience feel tailored without fragmenting the team's standards.

The strongest Claude Code setups treat CLAUDE.md as the constitution and memory as case law — one defines the rules, the other records how they've been interpreted and applied in practice.

## Frequently Asked Questions

### Can Claude Memory override CLAUDE.md instructions?

No. CLAUDE.md takes precedence for project-scoped behavior. If CLAUDE.md says "always run tests before committing" and a memory says "user prefers skipping tests," Claude follows CLAUDE.md. Memory influences communication style and decision-making but cannot override explicit project rules defined in CLAUDE.md.

### Do I need to set up Claude Memory manually?

No. Claude Memory is fully automatic — Claude creates, updates, and organizes memory files based on your conversations. You can ask Claude to remember specific things, and you can review or delete memory files in `.claude/projects/*/memory/`, but no manual setup is required. CLAUDE.md, by contrast, must be authored by hand.

### Should I check memory files into git?

No. Memory files are personal to you and stored in `.claude/`, which is typically gitignored. They contain user-specific preferences and conversation-derived context that wouldn't be appropriate to share with the team. CLAUDE.md is the file designed for git — it's the shared, team-reviewed source of project instructions.

### How do CLAUDE.md and skills files (SKILL.md) relate?

CLAUDE.md defines project-wide rules and context. SKILL.md files define task-specific workflows — how to write tests, generate content, or review code. Think of CLAUDE.md as "how we work on this project" and SKILL.md as "how we do this specific type of task." Both are checked into git and shared with the team; neither is personal like memory.

### What happens if CLAUDE.md and memory contradict each other?

Claude treats CLAUDE.md as authoritative for project behavior. If a memory conflicts with CLAUDE.md — for example, a memory recalls a build command that CLAUDE.md has since changed — Claude follows CLAUDE.md and should update or discard the stale memory. Claude is instructed to verify memory against current state before acting on it.
