**[Claude Code](/blog/claude-code-complete-guide)** is Anthropic's terminal-based AI coding agent that runs on your local machine. Its subagent system is a first-class feature: the `Agent` tool lets you spawn specialized sub-agents with distinct capabilities, contexts, and isolation levels. Unlike Codex's implicit parallelism, Claude Code's subagents are explicitly typed — you choose between `Explore` (fast read-only search), `Plan` (architecture design), `codex:codex-rescue` (delegation to Codex), and general-purpose agents.

Claude Code subagents operate within the same conversation context. You can run them in the foreground (blocking until results return) or background (continuing work while they execute). The `worktree` isolation mode creates a temporary git worktree so the subagent works on an isolated copy of the repo, preventing conflicts with your main working tree. This local-first approach gives you fine-grained control over orchestration but uses your machine's resources.

## Agent Architecture: Detailed Analysis

**Codex takes a serverless-function approach to multi-agent work.** Each task you submit becomes an independent execution unit. Codex clones your repo, sets up the environment (installing dependencies, running setup commands you've configured), executes the task, and produces a result — typically a git branch with changes. The agent has no awareness of other concurrent agents working on the same codebase.

This architecture has clear strengths for batch operations. Submit twenty bug reports and Codex processes them in parallel across twenty sandboxes. Each gets a clean environment, eliminating the risk of one task's changes interfering with another. The tradeoff is coordination: if task #7 depends on the output of task #3, you need to handle that sequencing yourself.

**Claude Code's subagent system is conversational and hierarchical.** The parent agent spawns child agents with explicit prompts, and the children return results to the parent. The parent can then synthesize findings, make decisions, and spawn additional agents. This enables multi-step workflows like: spawn an Explore agent to find all files referencing a deprecated API → analyze the results → spawn parallel agents to update each file → spawn a test runner to verify changes.

The typed agent system is where Claude Code pulls ahead on flexibility. An `Explore` agent is optimized for fast, read-only codebase search — it has access to Glob, Grep, and Read tools but cannot edit files. A `Plan` agent designs implementation strategies before any code is written. The `codex:codex-rescue` agent actually delegates work to OpenAI's Codex through a shared runtime, letting you use both platforms in a single workflow. This composability means you can build agent pipelines tailored to specific tasks rather than relying on a single general-purpose agent.

For a deeper look at how Claude Code's agent teams work in practice, see our coverage of [Claude Code Agent Teams](/blog/claude-code-agent-teams).

## Custom Agent Configuration: Detailed Analysis

**How you customize agent behavior differs significantly between the two platforms**, and this shapes how teams standardize their AI-assisted workflows.

Codex supports custom agent configuration primarily through system-level instructions. You can define an `AGENTS.md` file or equivalent configuration that tells Codex how to approach tasks — coding standards, testing requirements, PR conventions. These instructions apply uniformly to every task Codex processes. The Codex CLI also accepts flags and configuration for environment setup, allowing you to specify which commands to run before task execution (dependency installation, database migrations, etc.).

Claude Code's configuration system is more granular. At the project level, `CLAUDE.md` files define global instructions — quality gates, commit conventions, architectural constraints. But the [SKILL.md system](/blog/claude-code-extension-stack-skills-hooks-agents-mcp) goes further: you can create reusable instruction sets for specific task types. A `skills/test-writer/SKILL.md` file might define how to generate tests, while `skills/security-review/SKILL.md` defines a different workflow for security audits. When spawning a subagent, you can point it at a specific skill, giving each agent in your multi-agent workflow its own specialized instructions.

Claude Code also supports **hooks** — deterministic shell commands that execute in response to agent events. A pre-commit hook can run linters before any subagent commits code. A post-tool hook can log every file edit. This programmable layer sits outside the LLM, providing guarantees that no amount of prompt engineering can match. Read more about this in our guide to [Claude Code Hooks](/blog/claude-code-hooks-a-complete-guide-to-automating-your-ai-coding-workflow).

The practical difference: Codex gives you a consistent agent you configure once and deploy broadly. Claude Code gives you an agent toolkit where each subagent can be configured independently for its specific role.

## Workflow Integration: Detailed Analysis

**The way each tool fits into your development workflow determines which one you'll reach for on a daily basis.**

Codex integrates with GitHub pull request workflows natively. Each completed task produces a branch and PR that goes through your normal review process. You can submit tasks from the ChatGPT web UI, the Codex CLI, or the [VS Code extension](/blog/codex-vscode). For teams already using GitHub-centric workflows, this is seamless — Codex tasks appear as PRs alongside human-authored code. The async nature means you can queue up work at the end of the day and review results the next morning.

Claude Code integrates at the terminal level. Subagents execute within your active development session, which means you can interleave human and agent work fluidly. Start a refactoring task, spawn a background subagent to update tests while you continue editing, then review the subagent's changes before committing. The `SendMessage` capability lets you communicate with running subagents, redirecting their work mid-stream.

For CI/CD integration, Codex has a slight edge — its cloud-based architecture means it can run in environments where you don't have a local machine (mobile review, automated triage). Claude Code recently added [remote session support](/blog/claude-code-remote-sessions-phone), letting you launch and monitor sessions from your phone, but the execution still happens on your development machine or server.

## Isolation and Safety: Detailed Analysis

**Both platforms take isolation seriously, but their mechanisms reflect their architectural differences.**

Codex runs every task in a fully containerized sandbox with network access disabled by default. The agent cannot reach external services, install arbitrary packages from the internet, or affect your production systems. This makes Codex inherently safer for untrusted or experimental tasks — the blast radius of any single agent is confined to its container. When the task completes, the container is destroyed; only the git diff survives.

Claude Code's isolation is opt-in and more nuanced. The `worktree` isolation mode creates a temporary git worktree, giving the subagent its own copy of the repo to modify without affecting your working tree. But the subagent still runs on your local machine with the same permissions as your user account. Claude Code's permission system provides guardrails — tools require user approval based on configured permission modes — but it's a cooperative safety model rather than a containment model.

For teams working with sensitive codebases, Codex's container isolation is a stronger guarantee. For developers who need subagents to interact with local services (databases, running dev servers, API endpoints), Claude Code's local execution is a practical necessity that container isolation would break.

## When to Choose OpenAI Codex

**Choose Codex when you need to parallelize independent tasks at scale.** Its cloud-based sandbox architecture is purpose-built for scenarios where you have a backlog of well-defined tasks — fix these twenty bugs, write tests for these fifteen modules, update documentation across these ten repos.

Codex excels when:

- **Tasks are independent**: Each task stands alone without needing results from other tasks
- **You want async workflows**: Submit work, do something else, review results later
- **Security isolation matters**: The sandboxed environment prevents any task from affecting your system
- **Your team is GitHub-centric**: PRs are the natural output unit, and Codex produces them natively
- **You're on ChatGPT Pro or Team**: Codex is included in these plans, making the marginal cost of additional tasks low

OpenAI has also launched [Codex for open-source maintainers](/blog/codex-for-open-source) and [Codex for students](/blog/codex-for-students) with free credits, making it accessible for non-commercial use.

## When to Choose Claude Code

**Choose Claude Code when your multi-agent workflow requires coordination, context, and control.** Its typed subagent system and local execution model are designed for orchestrated sessions where agents need to share context and build on each other's work.

Claude Code excels when:

- **Tasks are interdependent**: Subagent B needs the findings from Subagent A before it can proceed
- **You need specialized agents**: Different phases of work benefit from different agent configurations (search vs. plan vs. implement vs. review)
- **Real-time orchestration matters**: You want to direct agents mid-stream, not just queue and wait
- **Local environment access is required**: Subagents need to interact with your dev server, database, or local APIs
- **You want programmable workflows**: Hooks, skills, and typed agents let you build repeatable multi-agent pipelines

The [Agent SDK](/glossary/agent-sdk) and skill system make Claude Code particularly strong for teams that want to codify their engineering practices into reusable agent configurations. See how teams are using these capabilities in [5 Claude Code Skills I Use Every Single Day](/blog/5-claude-code-skills-i-use-every-single-day).

## Verdict

**For batch, async, independent tasks: choose Codex.** Its cloud sandbox model scales horizontally without consuming local resources, and its GitHub-native output fits cleanly into existing review workflows. If you have twenty bug reports to triage or fifty files that need the same refactoring pattern, Codex processes them in parallel without breaking a sweat.

**For orchestrated, context-rich, multi-step workflows: choose Claude Code.** Its typed subagent system, conversation-level context sharing, and programmable hooks give you the control needed for complex engineering sessions where agents must coordinate. The ability to mix Explore, Plan, and implementation agents in a single workflow — and even delegate to Codex via the `codex:codex-rescue` bridge — makes Claude Code the more versatile orchestration platform.

Many teams will find the most productive approach is **using both**: Claude Code as the local orchestrator that delegates batch work to Codex when appropriate, keeping the strengths of each platform in play. ## Frequently Asked Questions

### Can you use Codex subagents inside Claude Code?
Yes. Claude Code includes a `codex:codex-rescue` agent type that delegates tasks to OpenAI's Codex through a shared runtime. This lets you use Codex's cloud sandbox for specific subtasks while Claude Code handles orchestration and context management locally.

### How many subagents can Claude Code run in parallel?
Claude Code can spawn multiple subagents simultaneously — you send parallel `Agent` tool calls in a single message. The practical limit depends on your machine's resources and API rate limits rather than a hard platform cap. Background agents run concurrently with your main session.

### Does Codex support typed or specialized agents like Claude Code?
Codex currently uses a single general-purpose agent type for all tasks. You can customize behavior through system prompts and configuration files, but there is no equivalent to Claude Code's typed agent system (Explore, Plan, code-reviewer) where different agents have different tool access and capabilities.

### Which platform is better for enterprise teams?
It depends on your workflow. Codex's container isolation and GitHub-native PRs fit teams with strict security requirements and PR-based review processes. Claude Code's skill system and hooks fit teams that want to encode engineering standards into programmable agent configurations. Both offer team/enterprise plans with appropriate access controls.

### Is there a cost difference between running subagents on each platform?
Codex tasks are included in ChatGPT Pro ($200/month) and Team plans with usage limits. Claude Code charges per-token via Anthropic's API, so subagent costs scale with the amount of context and generation each agent performs. For high-volume batch work, Codex's bundled pricing may be more predictable; for targeted, context-heavy sessions, Claude Code's per-token model can be more efficient.
