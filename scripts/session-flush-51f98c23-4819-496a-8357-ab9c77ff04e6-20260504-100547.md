## 🎓 MODEL LITERACY

**Tool-Input Repair Layers**: Today's story about DeepSeek outperforming Opus 4.7 reveals a truth most developers miss — when a model "fails" at tool calling, the real culprit is usually malformed JSON, missing required fields, or type mismatches between what the model outputs and what the tool schema expects. A tool-input repair layer sits between the model's raw output and the tool executor, validating the JSON against the schema and fixing common errors (wrapping bare values in objects, coercing types, filling defaults) before execution. The result: a $2 open model with a repair layer can outperform a $60 frontier model running without one. Before you upgrade your model, upgrade your plumbing.
