**SKILL.md files** define task-specific instruction sets. A `skills/security-review/SKILL.md` might include vulnerability patterns to check, secure coding standards to enforce, and a specific output format. When invoked, these skills inject domain expertise into the agent's context. Skills are reusable across team members because they live in the repository.

**Typed agent definitions** allow you to specify which tools an agent can access, what model it uses, and how it should behave. The `subagent_type` parameter selects from built-in types (Explore, Plan, code-reviewer) or custom definitions. This means you can create agents that are structurally constrained — a read-only research agent literally cannot edit files, reducing the risk of unintended changes.

**The Agent SDK** enables building entirely custom agent architectures. Beyond the built-in agent types, teams can define new agent types with specific tool access patterns, model configurations, and behavioral constraints. This is the foundation for building production-grade [multi-agent teams](/blog/claude-code-agent-teams) that mirror your engineering org's actual review and development workflows.

The practical impact: in Claude Code, you can create a pipeline where a Plan agent designs the approach, multiple Explore agents research the codebase in parallel, a general-purpose agent implements the changes, and a code-reviewer agent validates the output — all within a single session, orchestrated by the parent agent.

## Parallel Execution and Scale: Detailed Analysis

Both platforms support running multiple agents simultaneously, but the scaling characteristics differ because of their architectural choices.

### Codex: Cloud-Native Parallelism

Codex's cloud-based model means parallelism scales with your compute budget, not your local machine. Launch ten tasks, twenty tasks, fifty tasks — each gets its own container. Your laptop's CPU and memory are irrelevant because all execution happens server-side.

This makes Codex particularly strong for batch operations. Migrating a hundred files from one API pattern to another? Launch a Codex task per file (or per module) and let them all run simultaneously. The tasks don't need to coordinate because each migration is independent.

The async nature is also a workflow advantage. You can queue up tasks before lunch and review pull requests after. Codex tasks don't require your terminal to stay open or your machine to stay awake. Results persist in the Codex UI and can be reviewed, approved, or rejected at any time.

### Claude Code: Local Orchestration with Smart Delegation

Claude Code's parallelism is bounded by your local machine's resources and the model's API throughput. In practice, this means you can comfortably run 3-8 background subagents simultaneously before context management becomes a bottleneck.

The advantage is orchestration quality. Because subagents return results to a parent that understands the full context, Claude Code can do things Codex cannot: decompose a complex task, have multiple agents research different aspects, synthesize findings, and then execute a coordinated plan. This is not parallel-for-throughput — it's parallel-for-intelligence.

For large-scale batch work, Claude Code's worktree isolation allows multiple agents to make code changes in parallel without conflicts. Each agent works in its own worktree branch, and the parent can merge results afterward. This is less scalable than Codex's container model but provides tighter coordination.

## When to Choose OpenAI Codex

Codex is the stronger choice when your multi-agent needs are **task-parallel and independent** — meaning each agent's work doesn't depend on another agent's output within the same workflow.

**Best scenarios for Codex subagents:**

- **Bulk bug fixes**: You have 15 similar issues and each fix is self-contained. Launch 15 Codex tasks, review 15 PRs.
- **Large-scale migrations**: Updating API calls, refactoring patterns, or applying code mods across many files where each file is an independent unit of work.
- **Async team workflows**: You want to queue tasks for overnight execution without keeping a terminal open. Codex tasks persist and deliver results regardless of your local machine state.
- **CI/CD integration**: Codex's cloud execution model integrates naturally with pull-request-based workflows. Each task produces a PR that goes through your normal review process.
- **Resource-constrained local environments**: If your development machine is limited, Codex offloads all compute to the cloud. The [VS Code extension](/blog/codex-vscode) provides a lightweight local interface to a powerful remote execution environment.

Codex is less ideal when tasks need to share context mid-execution or when the decomposition of a complex task requires real-time coordination between agents.

## When to Choose Claude Code

Claude Code is the stronger choice when your multi-agent needs involve **orchestration, specialization, and context-dependent coordination** — meaning agents need to communicate, share findings, or adapt based on each other's results.

**Best scenarios for Claude Code subagents:**

- **Complex refactoring**: The parent agent plans the refactoring, Explore agents map dependencies, implementation agents make changes across files, and a reviewer agent validates the result — all in one coordinated session.
- **Codebase investigation**: Spawn multiple Explore agents to search different parts of the codebase simultaneously, synthesize findings, and act on the combined intelligence. This is how you answer "where is this pattern used and what would break if we changed it?" efficiently.
- **Custom review pipelines**: Define specialized agent types — security reviewer, performance analyzer, documentation checker — and run them as part of a structured review workflow. See how to build these in our [Claude Code complete guide](/blog/claude-code-complete-guide).
- **Iterative development with feedback loops**: The parent agent implements a feature, spawns a test-runner agent, gets results, fixes failures, and re-tests — all without human intervention between steps.
- **Team-specific workflows**: Use SKILL.md files and the Agent SDK to encode your team's exact development practices into reusable agent configurations. A "frontend feature agent" might run different checks than a "backend API agent."

Claude Code is less ideal for fire-and-forget bulk work where you want to queue dozens of independent tasks and walk away.

## Developer Experience Compared

Beyond architecture, the day-to-day experience of working with subagents differs in important ways.

**Setting up custom agents**: In Codex, you write an `AGENTS.md` file and a setup script — straightforward but limited to instructions and environment. In Claude Code, you can start simple with `CLAUDE.md` and scale up to typed agents with the Agent SDK. The learning curve is steeper but the ceiling is higher.

**Debugging agent behavior**: Codex provides execution logs and terminal output from each task's container. Claude Code shows subagent results inline in your terminal session, and you can resume a subagent with additional context using SendMessage. Claude Code's approach makes iterative debugging easier; Codex's approach makes post-hoc review of independent tasks cleaner.

**Cost model**: Codex charges compute credits that cover both cloud infrastructure and model tokens — the cost scales with task duration and container resources. Claude Code charges only for API tokens — your local machine provides the compute. For high-volume parallel work, Codex's costs can escalate quickly. For complex but fewer orchestrated workflows, Claude Code's token-only billing is typically more economical.

**Security model**: Codex's sandboxed containers have no access to your local environment, credentials, or network — by design. This is an advantage for sensitive codebases where you don't want AI agents running arbitrary commands on a developer's machine. Claude Code runs locally with the same permissions as the user, which provides more power but requires more trust. Permission modes and hooks provide guardrails, but the isolation is less comprehensive than Codex's container model.

## Verdict

**For orchestrated, context-dependent multi-agent workflows, choose Claude Code.** Its typed subagent system, background agents with notifications, worktree isolation, and Agent SDK give you a genuine agent orchestration platform. If your work involves complex tasks where agents need to share findings, specialize by role, and coordinate execution, Claude Code's architecture is purpose-built for this. The combination of [skills, hooks, agents, and MCP](/blog/claude-code-extension-stack-skills-hooks-agents-mcp) creates a programmable platform that goes well beyond basic code generation.

**For high-volume, independent parallel tasks, choose Codex.** Its cloud sandbox model scales horizontally without taxing your local machine, handles async workflows naturally, and provides strong isolation guarantees. If your use case is "run N independent coding tasks simultaneously and review the results," Codex's architecture delivers.

Many teams will use both. Codex for bulk migrations and independent bug fixes queued overnight. Claude Code for complex refactoring sessions, codebase investigations, and custom review pipelines during active development.

## Frequently Asked Questions

### Can Codex subagents communicate with each other during execution?

No. Each Codex task runs in an isolated container with no inter-task communication channel. If Task A's output should inform Task B, you must manually orchestrate them sequentially — run A, review its output, then launch B with that context included in the prompt.

### How many Claude Code subagents can run in parallel?

There is no hard limit enforced by the platform, but practical constraints include your machine's resources and API throughput. Most workflows run 3-8 background subagents comfortably. Beyond that, context management overhead increases and the parent agent may struggle to synthesize too many concurrent results effectively.

### Is Codex or Claude Code cheaper for multi-agent workflows?

It depends on the pattern. Codex charges compute credits covering cloud infrastructure and tokens — costs rise with task count and duration. Claude Code charges only API tokens, with local compute being free. For many short parallel tasks, Codex can be expensive. For fewer but complex orchestrated sessions, Claude Code's token-only model is usually cheaper. Calculate based on your actual usage pattern rather than list pricing.

### Can I define custom agent types in Codex like I can in Claude Code?

Not currently. Codex uses a single general-purpose agent type that you customize via `AGENTS.md` instructions and setup scripts. You cannot create structurally distinct agent types with different tool access or model configurations. Claude Code supports typed agents through the Agent SDK, allowing truly differentiated agent roles within a single workflow.

### Do I need the Agent SDK to use Claude Code subagents?

No. Claude Code's built-in agent types (Explore, Plan, general-purpose) work out of the box without any SDK setup. The Agent SDK is for teams that want to define custom agent types beyond the built-in options — specialized reviewers, domain-specific analysts, or production agent pipelines.
