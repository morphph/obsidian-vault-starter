**CLAUDE.md** files define project-level context — architecture decisions, coding standards, dependency choices, and constraints. Every agent (parent and subagent) automatically reads these files, ensuring consistent behavior without repeating instructions.

**SKILL.md** files encode specific task patterns — how to write tests, how to generate content, how to review security. These are reusable instruction sets that travel with the repository. When you invoke a skill, Claude Code loads the relevant SKILL.md and follows its patterns. Read about effective skill design in our [9 principles for writing skills](/blog/9-principles-writing-claude-code-skills-i-use-every-single-day).

**Agent type definitions** specify which tools each subagent type can access. The Explore agent cannot edit files. The Plan agent cannot write code. These constraints are enforced by the system, not by prompt instructions, reducing the risk of a subagent exceeding its intended scope.

**Hooks** add deterministic automation layers — shell commands that execute before or after specific agent actions. A pre-commit hook can run linting. A post-edit hook can validate schema changes. These operate outside the AI's decision-making, providing guardrails that apply uniformly. Our [hooks guide](/blog/claude-code-hooks-a-complete-guide-to-automating-your-ai-coding-workflow) covers the full system.

This layered approach means that a new team member cloning the repository immediately gets the same agent behavior — the customization is in the codebase, not in individual prompt templates saved in someone's notes.

## Real-World Workflow Patterns

### Pattern 1: Large Refactoring Across Multiple Modules

**Codex approach**: Submit separate tasks for each module — "refactor auth module to use new session API," "refactor payment module to use new session API," "update integration tests." Each runs in parallel in its own container. Review and merge PRs sequentially to avoid conflicts. Total wall-clock time is fast (parallel execution), but merge integration requires manual coordination.

**Claude Code approach**: Parent agent spawns an Explore subagent to map all usages of the old API across the codebase. Based on findings, it spawns Plan agents to design the migration for each module. Then the parent executes changes sequentially (or in worktree-isolated subagents for parallel file edits), running tests after each module. One commit or PR captures the entire refactoring. Tighter coordination, but limited to what fits in a single session's context.

**Verdict**: Codex is better for refactoring where modules are truly independent. Claude Code is better when changes interact and require cross-module awareness.

### Pattern 2: Investigating and Fixing a Bug

**Codex approach**: Submit a task describing the bug with reproduction steps. Codex investigates in its sandbox and proposes a fix. If the first attempt misses, submit a follow-up task with additional context. Each attempt is a fresh start in a clean environment.

**Claude Code approach**: Parent agent spawns an Explore subagent to search for the bug's origin. Spawns another to check recent git history for related changes. Synthesizes findings, forms a hypothesis, implements the fix, runs tests — all within one interactive session where you can redirect the investigation in real time.

**Verdict**: Claude Code's interactive orchestration is significantly faster for debugging, where investigation and fix are tightly coupled. Codex's isolation helps when the bug involves environment-specific behavior that benefits from a clean sandbox.

### Pattern 3: Generating Tests for an Untested Module

**Codex approach**: Submit a task: "Write comprehensive tests for the billing module." Codex reads the module, generates tests, and runs them in its sandbox to verify they pass. Multiple tasks can cover different modules simultaneously.

**Claude Code approach**: Parent agent reads the module, spawns a Plan subagent to determine test strategy, then writes and runs tests iteratively — adding cases, running them, fixing failures, until coverage goals are met. The interactive loop means edge cases discovered during testing are handled immediately.

**Verdict**: Codex is efficient for broad test generation across many modules in parallel. Claude Code produces tighter test suites through its iterative approach but handles one module at a time.

## Frequently Asked Questions

### Does Codex have a formal subagent API like Claude Code?

No. Codex achieves multi-agent behavior through task-level parallelism — you submit multiple independent tasks, each running in its own sandboxed container. There is no parent-child agent hierarchy or typed subagent system. Claude Code's Agent tool, by contrast, provides explicit subagent spawning with typed roles and result synthesis within a single session.

### Can I define custom agent types in Codex?

Not through a built-in type system. Codex custom agents are effectively different prompt templates and environment configurations that you maintain. You customize behavior through setup scripts (environment) and task descriptions (behavioral instructions). Claude Code offers a structured system: agent type definitions control tool access, SKILL.md files encode behavioral patterns, and CLAUDE.md provides project-level context automatically.

### Which approach is safer for running AI agents on production code?

Codex's container isolation provides stronger safety guarantees by default — each task runs in a sandboxed environment with no network access, and a misbehaving agent cannot affect other tasks or your local machine. Claude Code subagents share the local workspace, though typed agents have scoped tool access (Explore agents cannot edit files) and worktree isolation is available for parallel edits.

### Can Claude Code subagents run asynchronously like Codex tasks?

Partially. Claude Code subagents can run in the background within a session, allowing the parent agent to continue other work while waiting for results. However, this is session-scoped — you need an active Claude Code session. Codex tasks are fully asynchronous and cloud-hosted, meaning they run independently of any local session and can complete hours after submission.

### What is the cost difference for multi-agent workflows?

Codex is included in ChatGPT Pro ($200/month) with usage quotas, making parallel tasks cost-effective within those limits. Claude Code charges per token via API billing, and subagent work consumes additional tokens within the session. For high-volume parallel workloads, Codex's flat-rate model may be more predictable. For targeted, coordinated workflows with fewer total tokens, Claude Code's usage-based pricing can be more efficient.
