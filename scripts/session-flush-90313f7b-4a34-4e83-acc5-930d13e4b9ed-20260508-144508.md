# Codex Subagents vs Custom Agents：多智能体编码的两条路径

**TL;DR：** **Subagents** 是 OpenAI Codex 内置的任务分解机制，适合将一个大任务自动拆分成并行子任务执行。**Custom Agents** 是用户自定义的专用智能体，通过预设指令和工具配置让 Agent 在特定领域持续复用。选哪个取决于你的需求：一次性复杂任务选 Subagents 自动拆解，重复性专业任务选 Custom Agents 固化流程。

## 概览：Codex Subagents

**Subagents** 是 Codex 平台在执行复杂编码任务时自动生成的子智能体。当主 Agent 判断一个任务涉及多个独立模块时，它会将任务拆分为若干子任务，分别交给不同的 Subagent 并行处理。每个 Subagent 在独立的沙箱环境中运行，拥有完整的代码读取、文件编辑和命令执行能力。

这种机制的核心价值在于**自动并行化**。开发者不需要手动规划任务分解策略——你只需描述最终目标，Codex 会自行决定是否需要拆分、如何拆分、以及如何合并结果。这对大型代码库的重构、跨模块 Bug 修复、批量测试生成等场景尤其高效。更多关于 Codex 整体架构的介绍，可参考我们的 [Codex 完全指南](/zh/blog/codex-complete-guide)。

## 概览：Custom Agents

**Custom Agents** 允许用户基于 Codex 平台创建具有特定身份、指令和工具集的专用智能体。与 Subagents 由系统自动生成不同，Custom Agents 由开发者主动定义——你可以指定它的系统提示词、可调用的工具、访问的代码范围，甚至约束它的输出格式。

Custom Agents 的典型用法包括：创建一个专门做代码审查的 Agent、一个只负责写单元测试的 Agent、或者一个按照团队编码规范做重构的 Agent。它的价值在于**可复用性和一致性**——同一个 Custom Agent 可以反复使用，确保每次产出符合预定标准。

## 功能对比

| 维度 | Subagents | Custom Agents | 优势方 |
|------|-----------|---------------|--------|
| **创建方式** | 系统自动生成 | 用户手动定义 | 各有所长 |
| **任务类型** | 一次性复杂任务分解 | 重复性专用任务 | 各有所长 |
| **并行能力** | 原生并行执行 | 单 Agent 串行 | Subagents |
| **可定制性** | 低（系统决定行为） | 高（完全用户控制） | Custom Agents |
| **配置成本** | 零配置 | 需要编写指令和选择工具 | Subagents |
| **一致性保证** | 每次拆分策略可能不同 | 固定指令确保行为一致 | Custom Agents |
| **适用规模** | 大型跨模块任务 | 聚焦单一职能 | 各有所长 |

## 自动化 vs 可控性：核心权衡

这是选择 Subagents 还是 Custom Agents 时最关键的考量维度。

Subagents 的设计哲学是**最大化自动化**。你给 Codex 一个复杂任务——比如「重构这个模块的所有 API 路由，同时更新对应的测试文件」——系统自动判断需要几个并行任务，自动分配，自动合并。开发者的角色是审查结果，而不是规划执行。

Custom Agents 的设计哲学是**最大化可控性**。你预先定义 Agent 的行为边界：它应该关注哪些文件，遵循什么规范，使用什么工具，输出什么格式。每次调用的行为高度可预测。

实际项目中，这两种模式常常互补。团队可以先用 Custom Agents 定义「代码审查 Agent」和「测试生成 Agent」等专用角色，再在需要大规模重构时让 Codex 的 Subagent 机制自动调度这些角色。这种组合思路与 [多智能体工作流](/zh/blog/con-u-pour-des-workflows-multi-agents)的最佳实践高度一致。

## 实际应用场景对比

两种 Agent 模式在不同场景下的表现差异明显。理解这些差异才能做出正确选择。

**Subagents 擅长的场景：**

- **大规模代码迁移**：将一个 JavaScript 项目迁移到 TypeScript，涉及上百个文件的类型标注、import 修改和测试更新。Subagents 自动按模块拆分，并行处理，比手动分配快数倍。
- **跨模块 Bug 修复**：一个 Bug 的根因在数据层，但影响了 API、前端和测试三个层面。Subagents 同时修复所有受影响层，减少遗漏。
- **项目初始化**：从零搭建一个完整项目结构——目录规划、依赖安装、基础配置、CI 设置——多个子任务天然可并行。

**Custom Agents 擅长的场景：**

- **团队编码规范执行**：定义一个遵循团队代码风格的审查 Agent，每次 PR 都用同一个 Agent 审查，确保规范一致性。
- **特定框架的代码生成**：创建一个专门生成 React 组件的 Agent，内置你的组件模板、测试模式和命名规范。
- **安全审计**：配置一个只关注安全漏洞的 Agent，限制它的工具集和审查范围，避免干扰正常开发流程。

Claude Code 的 [Agent Teams](/zh/blog/claude-code-agent-teams) 机制提供了类似的多智能体协作能力，但采用了不同的架构思路——通过 `Agent()` 工具在本地 spawn 子 Agent，而非在云端沙箱中运行。

## 与 Claude Code Subagents 的对比

理解 Codex 的 Subagents/Custom Agents 绕不开与 Claude Code 的 [Subagents 实践](/zh/blog/claude-code-subagents-examples)比较。两者在底层架构上有本质区别。

**Codex** 的 Agent 运行在云端沙箱中，每个 Agent 拥有独立的计算环境。这意味着更强的隔离性和安全性，但也意味着更高的延迟和资源消耗。Custom Agents 的配置通过平台 UI 或 API 完成，适合团队共享和管理。

**Claude Code** 的 Subagents 运行在本地终端，通过 `Agent()` 工具调用。开发者可以指定 Agent 类型（Explore、Plan、codex-rescue 等），每个 Agent 继承主会话的上下文但在独立线程中执行。这种模式的优势是低延迟和深度项目上下文集成，劣势是受限于本地机器的计算资源。

对于追求 [Agentic Coding](/zh/glossary/agentic-coding) 实践的团队来说，两种平台的 Agent 机制可以结合使用：Claude Code 做本地的快速迭代和探索，Codex 做云端的大规模并行任务。

## 什么时候选 Subagents

以下情况优先考虑 Subagents：

- 任务涉及 **3 个以上独立模块**的同时修改
- 你对 Codex 的自动拆分策略有信心，不需要精确控制每一步
- 追求 **执行速度**而非执行路径的可预测性
- 一次性任务，不需要反复使用相同的 Agent 配置

Subagents 的零配置特性让它成为快速完成大型任务的首选。但要注意：自动拆分的质量依赖于任务描述的清晰度。含糊的指令会导致 Subagent 之间的职责边界模糊，增加合并冲突的风险。

## 什么时候选 Custom Agents

以下情况优先考虑 Custom Agents：

- 任务具有 **高度重复性**——每天或每周都要执行类似操作
- 需要严格遵循特定的代码规范、输出格式或工具约束
- 团队多人使用，需要确保 **行为一致性**
- 对 Agent 的行为边界有明确要求（比如只读特定目录，只使用特定命令）

Custom Agents 的前期配置投入会被后续的复用效率回收。一个配置良好的 Custom Agent 相当于一个永远在线、永远遵守规范的虚拟团队成员。参考 [Agent SDK](/zh/glossary/agent-sdk) 了解更多关于自定义 Agent 构建的底层能力。

## 结论

**Subagents 和 Custom Agents 不是竞争关系，而是互补关系。** Subagents 解决「如何高效执行一个大任务」的问题，Custom Agents 解决「如何可靠地重复执行同类任务」的问题。成熟的 AI 编码工作流通常两者兼用：用 Custom Agents 定义团队的标准化 Agent 角色，在需要大规模任务分解时让 Subagent 机制自动调度。

对于刚开始接触 Codex 多智能体能力的团队，建议从 Subagents 入手体验自动化拆分的效果，再逐步构建适合自己工作流的 Custom Agents。关于 Codex 在开源项目中的应用，推荐阅读 [Codex for Open Source](/zh/blog/codex-for-open-source) 了解更多实战案例。

## 常见问题

### Codex Subagents 需要额外付费吗？

Subagents 的计费包含在 Codex 的整体用量中，不单独收费。每个 Subagent 的 token 消耗会计入总用量，并行执行的多个 Subagent 会同时计费。具体费率取决于你的 Codex 订阅计划。

### Custom Agents 的配置可以在团队间共享吗？

可以。Custom Agents 的配置（包括系统提示词、工具选择和行为约束）可以通过 API 导出和导入，也可以在团队的 Codex 工作区中直接共享。这使得团队能够统一 Agent 行为标准。

### Subagents 和 Custom Agents 可以同时使用吗？

可以。Codex 允许在一个任务中同时使用两种模式。例如，你可以创建一个 Custom Agent 负责代码审查，然后在一个大型重构任务中让 Subagents 自动拆分执行，最后用同一个 Custom Agent 审查所有变更。
