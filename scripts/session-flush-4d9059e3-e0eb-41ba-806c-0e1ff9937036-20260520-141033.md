# Claude Code vs Codex：两大 AI 编程 Agent 深度对比

**TL;DR：** **Claude Code** 是终端原生的本地 Agent，实时读取你的代码库、执行命令、直接提交——适合需要即时反馈和深度项目上下文的开发者。**OpenAI Codex** 是云端异步 Agent，在沙盒环境里并行处理多个任务——适合想把大量独立任务批量甩出去、不需要盯着看的团队。选哪个取决于你的工作节奏：交互式还是异步式。

## 概览：Claude Code

**[Claude Code](/zh/glossary/agentic-coding)** 是 Anthropic 推出的 [Agentic Coding](/zh/glossary/agentic-coding) 工具，直接运行在你的终端里。它不是一个补全引擎，而是一个拥有完整 shell 访问权限的自主 Agent——能读取整个项目结构、规划多步骤任务、编辑跨文件代码、运行测试、提交 Git 变更。

Claude Code 的核心差异化在于它的[可编程扩展栈](/zh/blog/claude-code-extension-stack-skills-hooks-agents-mcp)：CLAUDE.md 定义项目级上下文，SKILL.md 编码可复用的工程规范，Hooks 实现事件驱动的自动化，MCP Server 对接外部工具。这套体系让 Claude Code 不只是一个"会写代码的 AI"，而是一个可以被团队标准化配置的工程基础设施。

定价采用按 token 计费的 API 模式，也可通过 Claude Pro/Max 订阅获得固定额度。

## 概览：OpenAI Codex

**OpenAI Codex** 是 OpenAI 于 2025 年推出的云端编程 Agent，基于 codex-1 模型（经过强化学习优化的编码专用模型）。与 Claude Code 最大的区别在于：Codex 的每个任务都在云端沙盒环境中执行，你提交任务后可以去做别的事，完成后收到通知再回来 review。

Codex 的异步特性使其天然支持并行——你可以同时派出多个任务，每个都在独立的沙盒里运行，互不干扰。它会自动克隆你的 repo、安装依赖、执行变更，最终生成一个 PR 或 diff 供你审核。[完整的 Codex 架构解析](/zh/blog/codex-complete-guide)可以帮你理解其工作原理。

ChatGPT Pro/Team/Enterprise 用户可直接使用，也可通过 [VS Code 扩展](/zh/blog/codex-vscode)在 IDE 中调用。

## 架构与工作流：根本性的差异

两者最核心的区别不在于模型能力，而在于**执行架构**。

Claude Code 是**本地优先、同步交互**的。它运行在你的机器上，直接读取你的文件系统，拥有和你一样的环境访问权限。你下达指令后，它实时执行，你可以随时中断、追加指令、调整方向。这种模式的优势是上下文极其完整——它看到的就是你真实的开发环境，包括未提交的改动、本地配置、正在跑的服务。

Codex 是**云端优先、异步执行**的。每个任务被派发到一个独立的云端沙盒，环境从你的 repo 快照克隆而来，没有网络访问（安全隔离设计）。这意味着 Codex 无法调用外部 API、无法访问私有 registry、无法跑需要网络的集成测试。但它的优势在于：你可以一次性派出 10 个任务，每个都在独立环境中并行执行，彼此不会冲突。

对于[多 Agent 协作](/zh/blog/claude-code-agent-teams)的场景，Claude Code 通过 Agent Teams 实现——主 Agent 派出子 Agent 在本地并行工作，共享同一个代码库上下文。Codex 的并行则是完全隔离的，每个任务看到的是 repo 的独立副本。

**判断标准：** 如果你的工作流是"一边写代码一边让 AI 帮忙"，Claude Code 的实时交互更合适。如果你的工作流是"早上列出 10 个 task 甩给 AI，自己去开会"，Codex 的异步模式更高效。

## 项目上下文与可配置性

开发者对 AI 编程工具最大的不满之一是"它不懂我的项目"。两款工具都试图解决这个问题，但方式截然不同。

Claude Code 的上下文系统是[七层可编程架构](/zh/blog/claude-code-seven-programmable-layers)：CLAUDE.md 定义项目全局规则，SKILL.md 编码特定任务的执行标准（比如"写测试时必须覆盖边界情况"），Hooks 在特定事件触发时自动执行脚本，MCP Server 连接外部数据源。这套体系的核心价值是**可版本化、可团队共享**——你把这些文件 check in 到 repo 里，团队每个人的 Claude Code 都遵循相同的规范。

Codex 使用 AGENTS.md 文件（类似 CLAUDE.md 的概念）来传递项目上下文，并支持在 setup script 中预装依赖和配置环境。相比之下，Codex 的可配置粒度更粗——它更像是"告诉 AI 项目背景"，而非"编程 AI 的行为"。

如果你需要精细控制 AI 的行为——比如[用 Hooks 做自动化质量门禁](/zh/blog/claude-code-hooks-mastery)、用 Skills 标准化团队的 AI 工作流——Claude Code 的扩展栈明显更成熟。如果你只需要"AI 理解我的 repo 然后干活"，Codex 的 AGENTS.md 足够用。

## 适用场景：选 Claude Code

- **交互式开发**：你在写功能，需要 AI 实时帮你调试、重构、补测试，整个过程是对话式的
- **复杂重构**：跨几十个文件的模块改名、架构迁移，需要 AI 理解完整上下文并保持一致性
- **需要本地环境的任务**：跑集成测试、访问本地数据库、调用内网服务——这些 Codex 的沙盒做不到
- **团队标准化**：通过 SKILL.md 和 Hooks 统一 AI 辅助的工程规范，适合中大型团队
- **持续对话**：任务之间有依赖关系，需要 AI 记住之前做了什么、接下来怎么做

## 适用场景：选 Codex

- **批量独立任务**：一次性甩出多个互不相关的 bug fix、feature 实现、测试补充，让 AI 并行跑
- **异步工作流**：提交任务后去做别的事，回来时 review 结果——适合管理者或技术 lead 分发工作
- **代码审查辅助**：让 Codex 对 PR 做自动 review 并生成修改建议
- **开源贡献**：Codex 的[开源计划](/zh/blog/codex-for-open-source)为开源维护者提供免费额度
- **教学场景**：[学生版 Codex](/zh/blog/codex-for-students) 提供 $100 免费额度，适合编程教学

## 定价与获取方式

Claude Code 的获取路径有三条：直接 API 调用（按 token 计费）、Claude Pro 订阅（$20/月，有速率限制）、Claude Max 订阅（$100-200/月，大幅提升额度）。重度用户通常选 Max 订阅以获得稳定的使用体验。

Codex 目前包含在 ChatGPT Pro（$200/月）、Team 和 Enterprise 套餐中。Pro 用户每月有固定的 Codex 任务额度。也可通过 API 单独调用。OpenAI 还为学生和开源维护者提供了专门的免费额度计划。

**费用判断：** 如果你已经是 ChatGPT Pro 用户，Codex 几乎是"附赠"的。如果你的主力模型是 Claude，直接用 Claude Code 更自然。跨平台使用两者在经济上通常不划算。

## 结论

**Claude Code 和 Codex 不是同一类工具的两个版本，而是两种不同工作范式的代表。** Claude Code 是你的"结对编程搭档"——实时在场、深度参与、可精细调教。Codex 是你的"异步任务队列"——批量派发、自动执行、回来验收。

如果你是每天写代码的工程师，需要一个随时能对话的 AI 搭档，**选 Claude Code**。如果你是技术 lead 或管理者，需要把大量标准化任务分发出去，**选 Codex**。两者并不互斥——不少团队正在探索[多 Agent 工作流](/zh/blog/con-u-pour-des-workflows-multi-agents)，将两者结合使用。

选择的关键不是"哪个更强"，而是"你的工作节奏是什么"。

## 常见问题

### Claude Code 和 Codex 能一起用吗？

可以。两者解决不同类型的问题——Claude Code 处理需要实时交互和深度上下文的任务，Codex 处理可以异步并行的独立任务。一些团队用 Claude Code 做核心开发，用 Codex 批量处理 bug fix 和代码审查。

### Codex 的沙盒环境不能联网，这影响大吗？

取决于你的项目。如果测试和构建完全依赖本地依赖（通过 setup script 预装），影响不大。但如果你需要跑集成测试、访问外部 API 或私有 registry，这是一个硬限制——这类任务更适合在本地环境运行的 Claude Code。

### 哪个工具对中文开发者更友好？

两者都支持中文交互。Claude Code 的 SKILL.md 系统允许你用中文编写项目规范和任务指令，对中文技术团队的工作流适配更灵活。Codex 的中文支持主要体现在对话层面，项目配置层面的中文化程度较低。
