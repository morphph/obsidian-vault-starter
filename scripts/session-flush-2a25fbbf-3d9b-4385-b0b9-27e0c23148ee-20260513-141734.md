# Claude Memory vs CLAUDE.md：两套记忆系统，该用哪个？

**TL;DR：** **Claude Memory** 是 Anthropic 为 Claude 设计的跨会话持久记忆，自动记住你的偏好、背景和工作习惯。**CLAUDE.md** 是 Claude Code 项目中的指令文件，用版本控制管理，为整个团队定义 AI 行为规范。两者不是竞争关系——**Memory 管"人"，CLAUDE.md 管"项目"**。大多数严肃开发者应该同时使用两者。

## 概览：Claude Memory

Claude Memory 是 Anthropic 在 Claude 产品线中内置的持久记忆系统，覆盖 Claude.ai 网页端、Claude Desktop 和 Claude Code。它解决的核心问题是：每次新对话都要重复交代背景信息。

Memory 的工作方式是在对话中自动或手动提取关键信息——你的角色、技术偏好、项目背景、协作习惯——然后以结构化条目存储。下次开启新对话时，相关记忆会被自动注入上下文，让 Claude 从"陌生人"变成"了解你的同事"。

2026 年初，Anthropic 对 Memory 进行了[重大升级](/zh/blog/anthropic-claude-memory-upgrades-importing)：免费用户也能使用记忆功能，并支持从 ChatGPT 导入历史数据。这意味着 Memory 已经从实验特性变成了核心基础设施。

在 Claude Code 中，Memory 以文件形式存储在 `~/.claude/projects/<project>/memory/` 目录下，包含 user（用户画像）、feedback（行为反馈）、project（项目动态）和 reference（外部资源指引）四种类型。

## 概览：CLAUDE.md

CLAUDE.md 是 Claude Code 的项目级指令文件，放在仓库根目录，随代码一起进入版本控制。它的定位是"项目的 AI 使用手册"——告诉 Claude Code 这个项目是什么、怎么构建、有什么约束、遵循什么规范。

CLAUDE.md 解决的问题与 Memory 完全不同：它不是关于"你是谁"，而是关于"这个项目怎么做"。构建命令、代码风格、架构决策、质量门禁、已知坑——这些信息属于项目而非个人，需要团队共享，需要代码审查，需要版本追踪。

CLAUDE.md 是 [Claude Code 七层可编程架构](/zh/blog/claude-code-seven-programmable-layers)的基础层。它还可以与 `skills/*/SKILL.md` 配合，前者定义全局规则，后者定义特定任务（如写测试、生成内容、做 code review）的专用指令。

## 功能对比

| 维度 | Claude Memory | CLAUDE.md | 优势方 |
|------|--------------|-----------|--------|
| **作用域** | 绑定用户账号，跨项目生效 | 绑定仓库，跨用户生效 | 各有所长 |
| **存储位置** | `~/.claude/` 用户目录 | 项目根目录，进 Git | CLAUDE.md |
| **版本控制** | 无版本历史 | 完整 Git 历史 | CLAUDE.md |
| **团队共享** | 仅个人可见 | 全团队共享 | CLAUDE.md |
| **写入方式** | 自动学习 + 手动触发 | 手动编写和维护 | Memory |
| **内容类型** | 用户偏好、反馈、项目动态 | 构建命令、代码规范、架构约束 | 各有所长 |
| **适用产品** | Claude.ai / Desktop / Code | 仅 Claude Code | Memory |
| **维护成本** | 低（自动积累） | 中（需要主动更新） | Memory |

## 核心差异详解：谁记什么

这是最容易混淆的地方。很多开发者第一次接触 Claude Code 的记忆体系时，会问："Memory 和 CLAUDE.md 不是都在'记东西'吗？为什么要两套？"

答案在于信息的归属权。

**Memory 记的是属于"你"的信息。** 比如你是后端工程师、偏好函数式风格、讨厌过度设计、commit message 喜欢用中文、上次让 Claude 别再每次回复末尾加总结。这些偏好跟着你走，不管你切到哪个项目都生效。如果你在 A 项目告诉 Claude"不要用 any 类型"，这条反馈会被记住，在 B 项目同样生效——因为它反映的是你的工程判断，不是某个项目的规则。

**CLAUDE.md 记的是属于"项目"的信息。** 比如这个仓库用 pnpm 不用 npm、构建前必须跑 lint、数据库用 SQLite 不用 Postgres、中文内容必须用 CJK 字数统计。这些规则跟着仓库走，不管谁来开发都必须遵守。新人 clone 仓库后，Claude Code 自动读取 CLAUDE.md，立刻了解项目约束——不需要老员工手动交代。

根据[Claude Code 记忆系统的设计理念](/zh/blog/claude-code-memory)，Memory 明确排除了应该由代码或 Git 历史推导出的信息（如文件路径、代码模式、架构），这些属于 CLAUDE.md 的职责范围。

## 实战场景：如何配合使用

理解了归属权的区别，配合使用就很自然了。

**场景一：新项目初始化。** 你创建一个新仓库，写好 CLAUDE.md 定义构建命令、测试策略和代码规范。Memory 中已经存着你的个人偏好（比如"commit message 用英文""不要加 emoji""解释代码时假设我有 10 年经验"），两者叠加后 Claude Code 既懂项目规则，又懂你的风格。

**场景二：团队协作。** 你和同事共享同一个仓库和同一份 CLAUDE.md，但各自有独立的 Memory。你让 Claude 写代码时不加注释，同事喜欢详细注释——Memory 让同一个项目里的 Claude 对不同人表现出不同风格，而 CLAUDE.md 保证所有人遵循同样的架构和构建流程。

**场景三：跨项目切换。** 你同时维护三个项目，每个有不同的 CLAUDE.md（不同的技术栈、不同的构建命令、不同的代码风格要求）。但你的 Memory 在三个项目间共享——你对 Claude 的使用偏好不需要重新建立。

**场景四：调试行为问题。** Claude Code 的表现不符合预期时，排查路径很清晰：如果是项目规则没被遵守（用了错误的包管理器、跳过了测试），检查 CLAUDE.md；如果是个人偏好没被尊重（又开始加总结了、又用了你不喜欢的命名风格），检查 Memory。

## 适用范围：不只是 Claude Code

一个关键区别是适用范围。CLAUDE.md 是 Claude Code 的专属机制，只在终端中的 [agentic coding](/zh/glossary/agentic-coding) 场景下生效。你在 Claude.ai 网页聊天时，CLAUDE.md 不会被读取。

Memory 则横跨整个 Claude 产品线。你在 Claude.ai 网页端积累的记忆，在 Claude Desktop 中同样可用。Claude Code 中的 Memory 以本地文件形式存在，但遵循相同的设计哲学：记住关于用户的信息，而非关于项目的信息。

这意味着如果你只用 Claude.ai 做日常问答和文档写作，Memory 是你唯一需要关心的记忆机制。CLAUDE.md 只在你使用 Claude Code 进行工程开发时才进入视野。

## 选型建议

**如果你是个人开发者：** 两者都用。CLAUDE.md 管项目规则，Memory 管个人偏好。初始投入不大——CLAUDE.md 写一次，Memory 自动积累。参考[如何高效地 prompt Claude Code](/zh/blog/how-to-effectively-prompt-a-claude-code) 建立最佳实践。

**如果你是团队负责人：** CLAUDE.md 是必须的——它是团队 AI 协作的基础设施，确保一致性。Memory 是可选但推荐的——让每个成员获得个性化体验。把 CLAUDE.md 纳入代码审查流程，像对待其他基础设施配置一样对待它。

**如果你只用 Claude.ai 不用 Claude Code：** 只需关注 Memory。它会自动工作，让你的对话体验随时间改善。

**如果你从 ChatGPT 迁移过来：** 先导入历史记忆（Anthropic 已支持），然后在你的开发项目中建立 CLAUDE.md。两步完成迁移。

## 结论

Claude Memory 和 CLAUDE.md 是互补的两层记忆架构，不存在"选 A 还是选 B"的问题。**Memory 是你的个人 AI 画像，CLAUDE.md 是项目的 AI 操作手册。** 前者让 Claude 记住你是谁，后者让 Claude 理解项目怎么做。想深入了解 Claude Code 的完整[扩展体系](/zh/blog/claude-code-extension-stack-skills-hooks-agents-mcp)，包括 Skills、Hooks、Agents 和 MCP 如何在 CLAUDE.md 之上构建更复杂的工作流，可以参考我们的专题分析。

## 常见问题

### Claude Memory 的内容会被用于模型训练吗？

根据 Anthropic 的隐私政策，Memory 中存储的信息用于改善你的个人体验，不会被用于训练基础模型。你可以随时查看、编辑或删除已存储的记忆条目。

### CLAUDE.md 支持多层级吗？

支持。Claude Code 会读取多个层级的指令文件：全局 `~/.claude/CLAUDE.md`、项目根目录 `CLAUDE.md`，以及子目录中的 `CLAUDE.md`。层级越深，优先级越高，适合在 monorepo 中为不同子项目设置不同规则。

### 两者冲突时谁优先？

CLAUDE.md 中的明确规则优先于 Memory 中的偏好。例如，如果你的 Memory 记录了"不写注释"，但项目 CLAUDE.md 要求"公共 API 必须有 JSDoc"，Claude Code 会遵循项目规则。项目约束大于个人偏好。
