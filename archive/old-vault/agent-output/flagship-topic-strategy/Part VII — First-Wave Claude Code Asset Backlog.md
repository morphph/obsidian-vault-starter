# Part VII — First-Wave Claude Code Asset Backlog

This backlog is designed to give Claude Code a concrete first publishing target after technical SEO stabilization. These are proposed priority assets for the first flagship-cluster completion wave.

## Backlog design rules

- Prioritize missing node types before expanding already-dense node types.

- Prefer assets that improve cluster completeness and capture clear search intent.

- Prefer assets that can be linked into the hub, cornerstone, and adjacent nodes immediately.

- Avoid creating multiple pages that target the same core intent.

## Notes on backlog conflicts

- Do not allow both a glossary-style "What Is Claude Code?" page and a cornerstone "What Is Claude Code? Complete Guide" page to compete for exactly the same intent unless architecture clearly differentiates them.

- If current topic-hub architecture already handles definitional intent well, use the cornerstone page as the primary head-term target and keep the glossary-style node either merged or deprioritized.

- If pricing intent becomes large enough, `claude-code-pricing` may deserve promotion from FAQ-style page to a richer standalone support page.
