# Tw93

## Summary
Chinese open-source developer (@HiTw93), creator of Kaku (terminal), Mole, Pake, MiaoYan. **Two viral Chinese-language Agent-engineering deep-dives** in our wiki: "你不知道的 Claude Code" (2026-03-12, 2.8M views, 17K bookmarks) and **"你不知道的 Agent：原理、架构与工程实践"** (2026-03-19, 1.69M views, 10.6K bookmarks). Both apply the same systematic deep-dive style. Together they're the most comprehensive Chinese-language coverage of Agent / Claude Code architecture in the wiki.

### Article 2 — "你不知道的 Agent：原理、架构与工程实践" (2026-03-19)
[[source-tw93-agent-architecture-engineering]] — 1.69M views, 10,590 bookmarks. The sequel.
- **Three judgment changes** he openly admits: (a) expensive models give less lift than expected; (b) Harness + verification quality > model quality; (c) "debug Agent" usually means "debug tool descriptions" — and the eval system itself is the hardest bug to find
- Anthropic [[agent-vs-workflow|Workflow vs Agent]] taxonomy + the **five canonical patterns** (Prompt Chaining / Routing / Parallelization / Orchestrator-Workers / Evaluator-Optimizer)

- **Counter-intuitive prompt-cache insight**: stable large system prompts cost *less* than frequently-varying small ones (90% cache discount vs full write price every time)
- **Skills design** (with hard data): `Use when / Don't use when` + anti-examples = 73% → 53% → **85%** accuracy. Three Skills anti-patterns.
- [[multi-agent-architecture|Multi-agent]] discipline: **protocol before collaboration** — JSONL inbox + `.tasks/` + `.worktrees/`
- [[agent-evaluation-traps|Evaluation traps]]: Pass@k vs Pass^k; environment / grader bugs masquerading as model regression; Anthropic airline-booking example (transcript vs outcome)
