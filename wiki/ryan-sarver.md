# Ryan Sarver

## Summary
VC and AI builder who constructed "Stella," an AI chief of staff on [[openclaw|OpenClaw]] that he claims surpasses any human chief of staff he's worked with. His system demonstrates production-grade [[zero-friction-capture]], [[kaizen-loop]], and the [[llm-judgment-vs-scripts]] design principle.

## Details
- Active VC: fundraising (100+ LP pipeline), sitting on boards, helping portfolio companies, angel investing
- Previously tried building AI EA as a product when first AI APIs came out — couldn't make it work. [[OpenClaw]] unlocked the capability.
- Built "Stella" — AI chief of staff with two-layer memory, meeting prep, task management, relationship tracking, email/calendar triage
- **Two-layer memory architecture:**
  - `memory/YYYY-MM-DD.md` — daily raw logs, auto-generated by scripts throughout the day
  - `MEMORY.md` — curated long-term memory, synthesized by Stella herself from daily notes, read on startup
- Chose **flat markdown over database** — human can read, edit, git-backup any memory file. No abstraction layer.
- **Operational rhythm:** 9am morning brief via WhatsApp, 6pm evening wrap, silence when nothing to report
- **Key design rule:** [[llm-judgment-vs-scripts]] — LLMs handle synthesis/prioritization/drafting, Python scripts handle deterministic work (file I/O, API calls, timestamps)
- Runs [[kaizen-loop]]: weekly research scan + Sunday review → continuous system improvement
- **Scale:** manages fundraise pipeline of 100+ LPs, relationship context on every person/company/project
