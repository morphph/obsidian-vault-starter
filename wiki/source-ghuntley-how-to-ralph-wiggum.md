# Source: How to Ralph Wiggum — Official Playbook

## Summary
Geoffrey Huntley's official methodology repository. Three-phase workflow (Requirements → Planning → Building), two prompts, IMPLEMENTATION_PLAN.md as disposable shared state. Advanced techniques: acceptance-driven backpressure, LLM-as-judge, JTBD → SLC releases.

## Source Details
- **URL**: https://github.com/ghuntley/how-to-ralph-wiggum
- **Author**: [[geoffrey-huntley|Geoffrey Huntley]]
- **Type**: GitHub repository / methodology guide

## Key Claims
1. Three phases: Requirements Definition → Planning Mode → Building Mode
2. Context is everything — spawn subagents rather than loading into main context
3. Specific prompt phrasing matters: "study" not "read", "don't assume not implemented"
4. Plans are disposable — regenerate when wrong
5. LLM-as-judge enables subjective quality criteria (tone, aesthetics)
6. JTBD → Story Map → SLC (Simple/Lovable/Complete) over MVP
